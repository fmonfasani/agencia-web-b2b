1️⃣ Discovery & Context

Objetivo: entender qué existe y qué problema se ataca.
Outputs: analysis/

Ejemplo:

Auditoría técnica del repo

Walkthrough del último sprint

Detección de deuda técnica

2️⃣ Planning & Design

Objetivo: decidir qué se va a hacer y cómo.
Outputs: proposal/ + implementation/

Ejemplo:

Propuesta de agregar analytics

Definir arquitectura mínima para backend

Plan de sprint con alcance cerrado

3️⃣ Task Decomposition

Objetivo: bajar el plan a trabajo ejecutable.
Outputs: tasks/

Ejemplo:

Día 1: setup GA4

Día 2: eventos de conversión

Día 3: tests E2E

Tags: frontend / backend / devops

4️⃣ Execution

Objetivo: construir.
Outputs: código + commits + PRs

Ejemplo:

Implementar tracking

Ajustar CI

Refactor chico detectado en el camino

5️⃣ Validation & Review

Objetivo: verificar que funcione y sirva.
Outputs: analysis/

Ejemplo:

QA manual

Lighthouse report

Walkthrough post-sprint

6️⃣ Documentation & Traceability

Objetivo: dejar evidencia y contexto futuro.
Outputs: todas las carpetas

Ejemplo:

Update de audit

Sprint walkthrough

Checklist marcado

7️⃣ Iteration / Evolution

Objetivo: decidir siguiente ciclo.
Outputs: proposal/ (V2)

Ejemplo:

Detectar necesidad de backend

Proponer evolución a ecosistema B2B
